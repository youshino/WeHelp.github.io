  • 關於這份申請網頁，分享一個開發時的技術心得。

    • 一開始碰到的瓶頸是學完 HTML 後，到 vscode 上卻不知該如何運用，

    這讓我了解到 重複實作 與 持續上網找資源 的重要性。


    這個申請書，我活用了 Selectors 概念中 “ class、id ” 屬性，以及 Grouping Selector 技巧，

    可以在相同 Element 中快速設定相同 Styling ，

    但每個名稱很凌亂，要如何 精準取名 是待改進的點。


    再來是設置連結網址，設定的過程讓我了解連結不只是點了就開啟，

    能利用 ” a :: selection ” 改掉預設的選取 Styling ，也能依 target 屬性決定開啟頁面。


    最後我還想到電腦、平板、手機不同螢幕尺寸的問題，

    我採 RWD 響應式設計 來處理螢幕縮小時， ” Table Tag 超出頁面範圍 ” 與 “ 大標題換行 ” 的問題，

    從中我需要精準指示 display : black & none 的位置，

    中間試錯的過程，訓練了我不斷找尋問題的思考力。


    最近也嘗試用剛學到的 JavaScrtpt 概念製作簡單的猜拳遊戲，

    從將 JS 設置的常數連結到 HTML 的資料開始，一步一步設變數、指令函數，

    讓我學習到一個函數能夠 隨著網頁需求 不斷增加 ，並非僅止於一次性指令完便定案。


    隨機出拳是用 “ Math.floor “ 包裹住 ” Math.random “ 來運算 “出拳圖示” 產生，

    當用 “ if “ 假設一個情況會發生後，

    也能加 “classList.add “ 新增一個 class 動作，比 CSS 多了邏輯運算，

    而我碰到的難題是，為了準確指令每個動作會執行什麼結果，

    後半階段設的常數非常相似，甚至只用大小寫區分，

    寫到後面容易忘記常數會執行到哪個函數，導致指定的值常常偵測錯誤，程式跑不動。

    之後若能 清楚記錄每個常數功能及執行動作 ，或許能減少許多錯誤。
