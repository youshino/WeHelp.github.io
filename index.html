  • 請描述一件產生明顯負面情緒的經歷，如何處理該情緒？

    • 其實我約略二月底時，才聽 Vivian 姐姐說有軟體工程師這條路，以及 WeHelp 這個訓練營，

    但在回答問題上， 表達自己想法 卻是一個困難點，

    我覺得我寫不出來申請書。


    我在製作申請書的過程中，為 “寫網頁” 、 ” Web 網絡概論 ” 、 ” HTML、CSS ” 下了許多功夫，

    每天的空閒時刻，幾乎不是在看教學影片、就是在寫程式或看理論書籍，

    但真的要面對彭彭老師問我們申請者的這些問題時，我的腦袋卻當機，

    畢竟我不是本科系，且碰到無法完成的事時，又容易焦慮、急躁，

    所以因為無法 將剛塞入腦袋的東西，馬上輸出成自己的言語跟想法，然後回答出來 ，

    使我感到非常焦躁。


    我除了學完 CSS flex 時，緊抓 Vivian 姐姐詢問接下來學習方向之外，

    寫這些問題時，我也會問她說：

    只學完這些真的夠申請嗎？、是不是應該做更多準備，才能更專業的回答問題？

    之類的話，但 Vivian 姐姐一直提醒我，彭彭老師是接受小白的，

    對自己有自信一點，未來就算要踏入“軟體工程師“這條路，

    也不能對自己猶豫，還是要不斷學習。


    所以我嘗試丟掉不必要的思考，繼續學習，

    且找了個簡單小遊戲跟著教學影片一起製作，

    透過做中學的過程，才慢慢了解這些問題並沒想像中那麼難。


  • 關於這份申請網頁，分享一個開發時的技術心得。

    • 我透過製作此申請書與猜拳遊戲，充分認知到 “ 細心 “ 的重要性，


    比如這個申請書，我為“ class、id ” 屬性取名稱時，取的很凌亂，

    導致想在相同 Element 中快速設定相同 Styling 時， CSS 看得眼花撩亂。

    還有在RWD 響應式設計中，我有時 black & none 的位置會放不對，

    導致電腦畫面同時出現兩個設定。


    製作猜拳遊戲時，我自取的常數名稱也常因類似性太高，

    導致指定的值常常偵測錯誤，程式跑不動。


    前端我認為最重要的就是細心，當繼續走到後端的邏輯運算技術，

    才能精準的下達指令，在複雜的程式碼中順利執行自己的目標成果。


  • 請用簡短兩三句話，描述對你而言，最重要的一項人際交往原則。

    • 我認為良好的溝通最重要，不僅能促進有效溝通，

    也更容易與他人建立更深層次的連結，解決問題並促進彼此理解。

    藉此建立令雙方舒適且良好的合作關係，一同達成共同目標。
