  • 如果參與這個訓練，會怎麼安排學習時間？

    1. 理解問題、釐清目標

    → 先了解此任務需要前進的方向。

    2. 上網查相關官網文檔

    → 自行找尋官方已統整好的資訊進行深入了解。

    3. 實作專案作業

    → 實作過程中若失敗，找出自己卡在哪，並另找能擺脫目前困境方案的資源，來解決現處問題。

  • 請描述一件產生明顯負面情緒的經歷，如何處理該情緒？

    • 其實我約略二月底時，才聽 Vivian 姐姐說有 WeHelp 這個跟軟體工程相關的訓練營，

    但在回答問題上，表達自己想法 卻是一個困難點，

    我覺得我寫不出來申請書。


    我在製作申請書的過程中，為 " 寫網頁 "、" Web 網絡概論 "、" HTML、CSS " 下了許多功夫，

    每天的空閒時刻，幾乎不是在看教學影片、就是在寫程式或看理論書籍，

    但真的要面對彭彭老師問我們申請者的這些問題時，我的腦袋卻當機，

    畢竟我不是本科系，且碰到無法完成的事時，又容易焦慮、急躁，

    所以因為無法 將剛塞入腦袋的東西，馬上輸出成自己的言語跟想法，然後回答出來 ，

    使我感到非常焦躁。


    我除了在學完 CSS flex 時，緊抓 Vivian 姐姐詢問接下來學習方向之外，

    寫這些問題時，我也會迷茫：

    只學完這些真的夠申請嗎？ 是不是應該做更多準備，才能更專業的回答問題？

    之類的，但我其實看過招生影片，WeHelp 訓練營是接受小白的

    而且如果未來有機會踏入 " 軟體工程師 " 這條路的話，

    根本沒時間讓自己猶豫，還是要不斷學習。


    所以我嘗試丟掉不必要的思考，繼續學習，

    且找了個簡單小遊戲跟著教學影片一起製作，

    透過做中學的過程，才慢慢了解這些問題並沒想像中那麼難。


  • 關於這份申請網頁，分享一個開發時的技術心得。

    • 在開發這份申請網頁時，我面臨了一項技術挑戰：

    如何在有限的開發週期內，實現這個看似不太難的申請書網頁。

    而在製作此申請書與猜拳遊戲的過程中，我也充分認識到 細心 的重要性。


    例如這個申請書，我為 " class、id " 屬性取名稱時，命名的很凌亂，

    導致想在相同元素中快速設定相同樣式時，CSS 看起來眼花撩亂。

    還有在 RWD 響應式設計中，有時 block 與 none 的位置會放不對，

    導致電腦畫面同時出現兩個設定，

    經過不停地檢查不同尺寸下的視覺效果、調整 CSS，

    這個問題才得以解決。


    我意識到細節才是關鍵，

    無論頁面結構多麼簡單，我都需要密切審視細節。

    因此，我將來會更加重視 HTML 標籤的運用與整體結構的檢視，

    並遵循 CSS 命名規範等，將 細心 內化為一種態度和習慣。
